# Domain Service Implementation Plan

## Overview
Implement a complete domain registration service for Clive as specified in the prompt. This includes RDAP availability checking, ResellerClub API integration, user-facing UI for search/purchase/management, and backend API routes. The service will support .com, .net, .org, and .co.za TLDs with real domain registration through ResellerClub.

## Prerequisites
- ResellerClub reseller account setup (human step)
- Environment variables configured (.env.local)
- Firebase/Firestore admin setup (already exists)

## Files to Create/Modify

### 1. Core Libraries (lib/domains/)

#### Update lib/domains/rdap.ts
- Replace existing content with prompt's RDAP implementation
- Add proper TLD extraction and validation
- Implement checkAllTLDs for parallel checking
- Handle unsupported TLDs and errors

#### Update lib/domains/resellerclub.ts  
- Replace with prompt's ResellerClub API wrapper

### 2. API Routes (app/api/domains/)

#### Create app/api/domains/check/route.ts
- GET endpoint for domain availability queries
- Support both specific domain and name-only searches
- Return results for all 4 TLDs

#### Create app/api/domains/pricing/route.ts
- Static pricing endpoint with ZAR cents
- Include register, renew, transfer, privacy costs

#### Create app/api/domains/register/route.ts
- POST endpoint for domain registration
- Live RDAP re-check before registration
- Create ResellerClub customer/contact if needed
- Write to Firestore, update user domain count

#### Create app/api/domains/list/route.ts
- GET endpoint for user's registered domains
- Return formatted domain data with timestamps

#### Create app/api/domains/renew/route.ts
- POST endpoint for domain renewal
- Update expiry in ResellerClub and Firestore
