# Plan: Enhance Striker Landing Page

## Goal
Transform the Striker landing page into a comprehensive, appealing destination with detailed technical specifications, enhanced appeal elements, and a dedicated competition registration flow. Make it more compelling for security researchers and hackers to engage with the product and competition.

## Current State
- Basic landing page with hero, how-it-works steps, downloads, pricing tiers, competition overview, and join steps
- Competition section has minimal details - just win condition, sandbox preview, prize tiers
- No detailed technical specs beyond basic features
- No appeal elements like testimonials or case studies
- Sign-up redirects to general /auth page

## Proposed Changes

### 1. Add Technical Specifications Section
- Detailed architecture explanation of Mahoraga engine
- How antibody system works (threat learning, adaptation cycles)
- Detection mechanisms and response types
- System capabilities and limitations

### 2. Enhance Competition Documentation
- Complete rules and regulations section
- Detailed allowed/prohibited actions (permitted tools, techniques, ethical guidelines)
- Competition timeline and format
- Judging criteria and verification process
- Legal disclaimers and liability waivers

### 3. Add Appeal Elements
- Placeholder testimonials from fictional security researchers (to be replaced with real ones)
- Sample success stories/case studies demonstrating Striker's effectiveness
- Community statistics (participants, antibodies created, etc.)
- Profiles of hypothetical past winners

### 4. Create Dedicated Competition Registration Page
- New route: /striker/competition/register
- Full sign-up flow requiring Clive account creation first
- Form for personal details, experience level, agreement to rules
- Integration with existing Clive auth system

### 5. Improve Overall Appeal
- More compelling copy and messaging throughout
- Social proof elements and statistics
- Clear value propositions for researchers
- Strategic call-to-actions

## Implementation Steps
1. Create placeholder content for testimonials, case studies, and winner profiles
2. Design and implement technical specifications section
3. Build comprehensive competition rules documentation
4. Develop competition registration page with form validation
5. Integrate new sections into existing landing page layout
6. Add appeal elements and improve copy throughout
7. Test complete user flow and responsive design

## Relevant Files
- app/striker/page.tsx: Main landing page (to be enhanced)
- app/striker/competition/register/page.tsx: New registration page
- components/striker/ArchitectureSection.tsx: New specs component

- components/striker/TestimonialsSection.tsx: New appeal component
