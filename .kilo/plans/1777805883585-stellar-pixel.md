# Fix Firebase Admin SDK Build Error in products.server.ts

## Problem
The Next.js build is failing with a TypeScript error: "Module 'firebase-admin/firestore' has no exported member 'collection'". This occurs because the code is attempting to import Firestore query functions (collection, query, where, etc.) from 'firebase-admin/firestore', but these functions are not exported in the Firebase Admin SDK v13+. Instead, Firebase Admin SDK uses instance methods on the Firestore object.

## Root Cause
- Firebase Admin SDK uses a different API pattern than the client SDK
- Functions like `collection()`, `query()`, `where()` are not exported from 'firebase-admin/firestore'
- The correct approach is to use chained methods on the Firestore instance (e.g., `db.collection('products').where(...)`)

## Solution Steps

1. **Update imports in lib/firebase/products.server.ts**
   - Remove imports of `collection`, `query`, `where`, `getDocs`, `getDoc`, `doc`, `limit` from 'firebase-admin/firestore'
   - Keep only necessary types like `DocumentData`

2. **Refactor Firestore operations to use instance methods**
   - Change `query(collection(adminDb(), 'products'), where('status', '==', 'live'))` to `adminDb().collection('products').where('status', '==', 'live')`
   - Change `getDocs(q)` to `q.get()` for queries
   - Change `doc(collection(adminDb(), 'products'), id)` to `adminDb().collection('products').doc(id)`
   - Change `getDoc(docRef)` to `docRef.get()`

3. **Fix snapshot property access**
   - Change `snap.exists()` to `snap.exists` (property, not method)
   - `snap.empty` remains as property (already correct)

4. **Verify build success**
   - Run `npm run build` to ensure TypeScript compilation passes
   - Check for any remaining type errors
